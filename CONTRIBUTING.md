# Contributing to Sello

Thanks for taking the time to contribute. This guide covers the essentials and
links to the full contributor documentation.

## Code of Conduct

By participating, you agree to follow the Code of Conduct in
`CODE_OF_CONDUCT.md`.

## Security Issues

Please do not report security vulnerabilities in public issues. Email
security@sello-project.org with a detailed report.

## Getting Started

1. Fork the repository and create a feature branch.
2. Follow the development setup in `docs/DEVELOPER_GUIDE.md`.
3. Make your changes with tests and documentation updates.
4. Run `cargo fmt --check`, `cargo clippy -- -D warnings`, and `cargo test`.
5. Open a pull request.

## Full Guidelines

See `docs/CONTRIBUTING.md` for the detailed workflow, PR checklist, commit
message format, and review process.
